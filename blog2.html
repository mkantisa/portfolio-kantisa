Clinic Appointment & Management System (Semester Project)

This project was developed as part of a semester group assignment where we built a clinic appointment and management system. The main goal was to understand how a real-world web system is planned, designed, and developed from scratch using a structured development approach.

The system focuses on simplifying clinic operations such as patient registration, appointment scheduling, and record management through a web-based platform built using CodeIgniter 4 (CI4) and MVC architecture.

Miss Kantisa

Student Developer | UI/UX & Web Design Learner

Getting Started: Why This Project Matters

This was a group-based semester project where we built a clinic appointment and management system as part of our web development course. The main idea behind the project was not just to complete an assignment, but to understand how a real system is planned, designed, and built from the ground up.

Unlike small practice projects, this one required us to think in terms of real users, real workflows, and real system structure — from how patients register, to how appointments are scheduled and managed by clinic staff.

Understanding the Real Problem

Before jumping into coding, we first analysed how a clinic actually operates in real life. Things like long queues, manual appointment booking, and miscommunication were some of the problems we wanted to simplify.

From there, the idea of a digital clinic system started to make sense — a platform where everything could be managed in one place with less confusion and better organisation.

System Planning Phase

This stage was actually one of the most important parts of the whole project. Before writing any code, we focused on system design and documentation.

We created diagrams such as Data Flow Diagram (DFD), Use Case Diagram, and Entity Relationship Diagram (ERD). These helped us visualise how data moves through the system and how each user interacts with it.

At first, it felt like extra work, but later it became clear that this step made the development process much smoother and more organised.

Database Design

After finalising the system flow, we designed the database structure. This part was crucial because everything in the system depends on how data is stored and connected.

The main tables included patients, doctors, appointments, and user accounts. Each table was linked using relational keys to ensure data consistency and proper system behaviour.

This helped us understand how real applications manage and structure large sets of data behind the scenes.

Development Using CodeIgniter 4

Once the planning and database structure were ready, we moved into development using CodeIgniter 4 (CI4). The MVC (Model-View-Controller) structure was used to organise the system properly.

This approach helped separate responsibilities — database logic in models, application logic in controllers, and user interface in views.

Some of the core features developed include appointment booking, patient management, and an admin dashboard for managing system data.

My Role and Contribution

During this project, I was mainly involved in frontend development and integrating the UI with the backend system. This included building responsive interfaces and ensuring the user experience remains simple and easy to use.

I also worked on connecting form inputs to backend functions so that data could be stored and retrieved properly from the database.

Challenges During Development

One of the biggest challenges was understanding how CI4 connects everything together — especially how data flows between controller, model, and view.

Another challenge was managing consistency across the system while different parts were being developed at the same time. Sometimes small changes in database structure affected multiple modules, which required careful adjustment.

What I Learned From This Project

This project gave me a much deeper understanding of how real web systems are built compared to smaller practice projects.

I improved my skills in several areas:

  • System analysis and requirement understanding
  • Creating DFD, ERD, and Use Case diagrams
  • Database design and relationships
  • Working with CodeIgniter 4 MVC structure
  • Building functional CRUD-based systems
  • Improving UI/UX consistency in real applications

Final Thoughts

Overall, this project was a turning point in my learning journey. It showed me that building a real system is not just about coding, but also about planning, structure, and understanding user needs.

Even though it was challenging at times, it helped me grow more confident in web development and gave me a clearer direction for future projects.

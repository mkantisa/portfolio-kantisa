Developing StudyLog: My Final Year Project Journey

This article shares my experience developing StudyLog, a web-based study management system created as my Final Year Project. The project focuses on helping students organise academic tasks, track study progress, and manage learning activities through a simple and user-friendly platform.

From requirement analysis and system design to implementation and testing, StudyLog has given me valuable experience in developing a more complete and structured web application while applying the knowledge gained throughout my degree programme.

Developing StudyLog: My Final Year Project Journey

StudyLog is my Final Year Project (FYP), developed to help students manage their academic activities in a more organised and structured way. The idea came from a common problem faced by many students — managing assignments, study schedules, deadlines, and personal learning progress across multiple subjects at the same time.

Unlike my previous projects, StudyLog required a more complete development process involving system analysis, requirement gathering, design documentation, implementation, testing, and evaluation. This project allowed me to apply the knowledge and skills I have gained throughout my degree programme into a single integrated system.

Identifying the Problem

Many students rely on notebooks, messaging applications, or multiple productivity tools to manage their academic work. As a result, important tasks and deadlines can sometimes be overlooked.

The objective of StudyLog is to provide a centralised platform where students can record study activities, manage tasks, monitor progress, and organise academic information more efficiently.

Before development began, I conducted research on existing study management applications and identified several features that could improve the overall learning experience for students.

System Analysis and Design

One of the most important stages of this project was the planning and analysis phase. Before writing any code, I prepared various system design documents to understand how the application should function.

These included Use Case Diagrams, Data Flow Diagrams (DFD), Entity Relationship Diagrams (ERD), system flowcharts, and functional requirements documentation. Creating these diagrams helped me visualise the entire system structure and identify relationships between different modules.

Through this process, I gained a better understanding of how software projects are planned before development begins.

Designing the Database Structure

After completing the analysis phase, I designed the database structure that would support the system. Since StudyLog manages multiple types of information, careful database planning was necessary to ensure that data could be stored and retrieved efficiently.

The database includes modules related to user accounts, study records, tasks, deadlines, progress tracking, and activity logs. Relationships between tables were designed to maintain data integrity while supporting future system expansion.

This stage strengthened my understanding of relational database concepts and database normalisation techniques.

System Development

Once the planning and database design were completed, development of the system began. The focus was on creating a clean, user-friendly interface while ensuring that the core functionality worked efficiently.

Several key features were implemented, including task management, study activity recording, progress tracking, and user account management. The system was designed to allow students to monitor their academic responsibilities in a single platform.

Throughout development, I continuously improved the interface design to make navigation simple and intuitive for users.

Challenges During Development

Developing StudyLog presented several challenges. One of the main difficulties was ensuring that all modules worked together smoothly while maintaining a structured system architecture.

Managing database relationships, implementing CRUD operations, and ensuring data consistency required careful attention throughout the development process.

Another challenge was balancing functionality and usability. While adding more features could increase system capabilities, it was equally important to keep the interface simple and easy to use.

Skills and Knowledge Gained

This project significantly improved both my technical and problem-solving abilities. Through the development of StudyLog, I gained valuable experience in:

  • System analysis and requirement gathering
  • Use Case Diagram, DFD, and ERD design
  • Database design and management
  • Frontend interface development
  • CRUD implementation and system functionality
  • Testing and debugging processes
  • UI/UX design principles
  • Project planning and documentation

More importantly, the project taught me how to approach software development in a structured and professional manner.

Final Reflection

StudyLog represents one of the most significant projects in my academic journey. It combines system analysis, design, development, and problem-solving into a single application that addresses a practical issue faced by students.

Through this Final Year Project, I have gained a deeper understanding of the software development lifecycle and strengthened my confidence in building complete web-based systems. The experience has also motivated me to continue improving my technical skills and explore more advanced technologies in the future.

Looking back, StudyLog is more than just a university project. It reflects my growth as a student developer and demonstrates how the knowledge gained throughout my studies can be applied to create practical solutions for real users.
